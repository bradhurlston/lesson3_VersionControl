What is Git? How does it work?

Git is a system that stores information much differently than any other system. It thinks about data in a different way. It thinks of it in "snapshots". This basically means that it's taking a look at the file at that moment. Everything git does only requires local files which makes operations happen in an instant at lighting fast speeds. One very important part of git and the way it works is that it's impossible to change a file without Git knowing about the change or corruption. It does this by using a mechanism called SHA-1 hash which is a string of characters that git stores everything in.

Three States of Git And Three Sections

It is important to know that git has three main states for files to reside within. The three states are committed, modified, and staged. The committed state stores data safely in a database. Modified deals with changing the file but no yet committing it. The staged state just means that a modified file was marked and is ready to move to your next commit snapshot. Along with the three states, there are three sections of git; the git directory, the working directory, and the staging area. The files are first modified in the working directory. They are then staged and saved in screenshots. Then after a commit, they are permanently stored in the git directory.

Version Control System (VCS)

Before knowing what a version control system is, one must know what version control is. Version control is one main repository for all project files. So what is a VCS and what does it do? This automatically recognizes who changed a file, when it was changed, and was about it was changed. In a VCS, files cannot be overwritten which allows people to work on the same file at the same time without any problem or conflict. There is a common repository that holds files. When a repository is created with files and directories, developers are able to record their change history and track their status and versions.

Changes to the Repository

When it comes to changes in the repository, there are four main kind of files. These include untracked, unmodified, modified, and staged. Tracked files either fall into the unmodified, modified, or staged category. Untracked however are anything else. These are files that were not in the staging area and were not in the last snapshot. When editing files, Git will see them as modified. This is because they’ve been changed since your last commit. You first stage these modified files and then commit all staged changes. This then leads to a cycle repeat.

Git Actions

At first, git actions are somewhat overwhelming as it's a lot to learn all at once. But like everything, it takes practice. There are eleven basic git commands that can be used consistently and should be known. The first basic commands are git commit, git branch, git checkout, and git checkout -b. The two undo commits are git reset and git revert. The combine branch commands are git merge and git rebase. The last git commands are for the remote server. These include git fetch, git pull, and git push. All these combine with the git states and stages are the basis of git.
